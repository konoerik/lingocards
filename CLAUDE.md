# CLAUDE.md

## Project Overview

LingoCards is a Progressive Web App (PWA) flashcard app for young children learning a new language. Cards show an image or emoji, the target-language word, a romanized pronunciation guide, and a translation in the learner's base language. Greek is the primary deck; the architecture supports adding other languages (Spanish, etc.) by adding an entry to `data/manifest.json` and a `data/decks/<lang>.json` file — no code changes needed.

## Tech Stack
- **Languages:** HTML, CSS, vanilla JavaScript
- **Libraries:** None — no CDN dependencies
- **Fonts:** Nunito via Google Fonts CDN
- **Build step:** None
- **Python scripts:** Pillow (icons), ElevenLabs + gTTS (audio) — run inside `.venv`

## Key Conventions
- Entry: `index.html` → `src/app.js` + `src/style.css`
- Deck registry: `data/manifest.json` (key, label, flag, enabled, user_notes per deck)
- Card content: `data/decks/<lang>.json` — adding/editing words requires no code changes
- Images go in `images/<category>/` — referenced by `image` field in the deck file
- Audio goes in `audio/` — generated by `scripts/generate_audio.py`
- Python scripts run inside `.venv` — always use `.venv/bin/python`, never global pip

## Development Workflow
```bash
# First-time setup
make install        # creates .venv and installs requirements.txt (Pillow, gtts)

# Serve locally (required — data files are fetched over HTTP)
make serve          # python3 -m http.server 8080

# Generate assets
make icons          # generate icon-192.png / icon-512.png (one-time)
make images         # generate card illustrations (requires OPENAI_API_KEY in .env)
make audio          # generate audio files (set ELEVENLABS_API_KEY in .env; falls back to gTTS)
```

## Testing & Visual QA
```bash
# One-time: install Playwright and download browser engines (~200 MB)
make install-dev

# Capture screenshots across all device viewports (no separate server needed)
make screenshots
```

Produces `screenshots/chromium/<device>.png` and `screenshots/webkit/<device>.png` for:
- **iPhone:** SE 3rd gen (375×667), iPhone 14 (390×844), iPhone 17 Pro Max (430×956)
- **Android:** Pixel 7 (412×915), Galaxy S24 (360×780), Galaxy A54 (360×800)
- **Tablet/desktop:** iPad 10th gen (820×1180), MacBook Air 13" (1280×800), Desktop 1440p

`webkit` uses Safari's rendering engine; `chromium` approximates Chrome. The script starts its own HTTP server on port 8765 so it can run standalone. `screenshots/` is gitignored.

## Behavior Rules
- No build step — if a library isn't available via CDN, discuss before adding a bundler
- Always use `.venv` for Python — never install packages globally
- When making a structural decision (adding a library, splitting into multiple pages), record it with `/log`
