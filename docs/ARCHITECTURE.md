# Architecture

## Quick Reference
**Stack:** HTML, CSS, vanilla JS — no build step
**Entry point:** `index.html`
**Styles:** `src/style.css`
**Logic:** `src/app.js`
**Libraries:** Nunito font (Google Fonts CDN only)
**Data:** `data/words.json` fetched at runtime; progress in localStorage
**Key constraint:** Must be served over HTTP (not file://) — use `python3 -m http.server 8080`

## Decisions (ADRs)
<!-- Append new ADRs with /log -->
- **No framework** — vanilla JS sufficient for this scope; eliminates build step and maintenance overhead
- **Greek-first, multi-language ready** — Spanish deck is empty array in words.json; language dropdown is in the UI but Spanish option is disabled until content is ready
- **Image-optional** — every card has an emoji fallback; `image` field is optional in words.json; `.has-image` CSS class controls which is shown
- **Leitner in localStorage** — simple 3-box SRS stored client-side; no backend needed
- **Settings behind 5-tap** — prevents child from opening settings accidentally


## Detail

### Page structure
- `<header class="topbar">` — language dropdown + settings gear
- `<nav class="category-tabs">` — scrollable tab row, populated by JS from deck categories
- `<main>` — card wrapper, progress bar, three answer buttons, empty state
- `<div class="settings-overlay">` — modal sheet, hidden by default

### JS organisation (`src/app.js`)
- **State** — `deck`, `progress`, `queue`, `currentIdx`, `language`, `category`, `settings`
- **`init()`** — loads settings + progress from localStorage, fetches words.json, wires listeners
- **`switchLanguage()` / `switchCategory()`** — update state, rebuild queue, re-render
- **`buildQueue()`** — filters deck by category, separates cards into Leitner boxes, orders by BOX_ORDER (Learning → New → Known), filters to due cards only
- **`renderCard()`** — populates card DOM from current queue position; shows empty state when queue exhausted
- **`answer(rating)`** — applies Leitner transition, saves progress, advances queue
- **Audio** — `playAudio()` plays voice_1 on auto-advance; `playAudioRandom()` plays a random voice on button tap
